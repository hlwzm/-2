# 指尖江湖2 - 开发并行分工方案

## 一、团队结构

制作人 (1人): 统筹全局/里程碑管理
  客户端主程 (1人): Unity架构/渲染/网络层
    UI组 (2-3人): 各系统界面实现
    战斗组 (2-3人): 战斗逻辑/技能/AI
    系统组 (2人): 背包/装备/任务/聊天
  服务端主程 (1人): Go架构/数据库设计
    网关组 (1人): 网关/登录/鉴权
    战斗服组 (2人): 战斗/副本/匹配
    业务组 (2人): 交易/聊天/社交/后台
  美术主美 (1人): 风格把控/资源规范
    角色组 (3-4人): 英雄建模/贴图/骨骼
    场景组 (3-4人): 地图/建筑/植被
    动作组 (2人): 角色动作/技能动作
    特效组 (2人): 技能特效/UI特效
  策划组 (3-4人): 系统/数值/文案
  测试组 (2-3人): 功能/性能/压力

## 二、并行开发阶段

### 第1梯队 (第1-4周) - 基建
- 组A(客户端): Unity框架 + URP管线 + UI框架 + 网络层
- 组B(服务端): 网关 + 登录 + 数据库搭建 + 消息队列
- 组C(美术): 风格定稿 + 首批英雄概念设计 + 场景概念设计
- 组D(策划): 数值框架 + 战斗公式 + 剧情大纲

### 第2梯队 (第3-8周)
- 组E: 英雄系统 + 招募(抽卡)系统 + 编队功能
- 组F: 背包 + 装备系统 + 强化/精炼
- 组G: 聊天系统 + 好友 + 私聊
- 组H(美术): 首批8名英雄建模 + 稻香村场景

### 第3梯队 (第6-12周)
- 组I: 战斗系统 + 技能系统 + 伤害计算
- 组J: 副本系统 (副本1+2 完整流程)
- 组K: 交易/拍卖行系统 (含5%手续费)
- 组L(美术): 第二批英雄 + 野外地图

### 第4梯队 (第10-16周)
- 组M: PVP系统 + 竞技场 + 匹配
- 组N: 商城 + 充值 + 月卡/战令
- 组O: 剧情任务系统 + 成就
- 组P: 同盟系统 + 组队系统完善

### 第5梯队 (第14-20周)
- 组Q: 后台管理系统 (Web界面)
- 组R: 性能优化 + 压测
- 组S: Bug修复 + 上线准备

## 三、Git分支策略
