# 指尖江湖2 — 系统设计指南（V1.0）

## 一、项目架构

### 1.1 目录结构
```
指尖江湖2/
├── Client/                    # Unity客户端
│   └── Assets/Scripts/
│       ├── Core/              # 核心框架
│       │   ├── Network/       # TCP网络 + 协议编解码
│       │   ├── Manager/       # 12个Manager（单例）
│       │   ├── Battle/        # 战斗引擎/HeroUnit/EnemyUnit/HeroSwitch
│       │   ├── Scene/         # 场景管理 + 6个SceneBoot
│       │   └── Config/        # Hero/Skill/Item/Dungeon配置表
│       ├── UI/
│       │   ├── Panels/        # 14个UI面板（全程序化）
│       │   └── Battle/        # BattleHUD/BuffIcon
│       ├── Battle/
│       │   └── Skill/         # 技能特效SkillEffect
│       └── Game/              # Bootstrapper入口
├── Server/                    # 微服务（C# .NET）
│   ├── Jx3.MockServer/        # 登录+进入游戏
│   ├── Jx3.Battle/            # 战斗结算
│   ├── Jx3.Chat/              # 聊天
│   ├── Jx3.Dungeon/           # 副本
│   ├── Jx3.Trade/             # 交易行
│   ├── Jx3.Shop/              # 商城
│   ├── Jx3.Social/            # 好友/帮会
│   ├── Jx3.PVP/               # PVP竞技
│   └── Jx3.Quest/             # 任务
├── Documents/                 # 设计文档
├── Tools/                     # 辅助脚本
└── deploy/                    # 部署配置
```

### 1.2 客户端核心类图
- Bootstrapper → GameManager(单例) → 所有Manager
- 每个Manager → 对应Panel(UI) + 对应Server(微服务)
- NetworkClient(单例) → TCP连接 → MockServer

### 1.3 通信协议
```
[4B长度前缀] [packet bytes]
Packet = magic(2) + msgId(4) + seq(4) + bodyLen(4) + flag(2) + body(N)
Magic = 0x9A7B
```
当前MockServer处理: MsgId 1001(登录) + 1007(进入游戏)

## 三、UI开发规范
- 所有面板必须由美术老师先设计UI再转交代码
- 使用全程序化生成（无Unity预制体）
- 继承BasePanel，调用base.Awake()
- 暗黑紫色主题：bg #0a0a14, card #12121e, accent #b088ff
- Font: Resources.GetBuiltinResource<Font>("LegacyRuntime.ttf")
- Image: new GameObject(name, typeof(RectTransform), typeof(Image))

## 四、启动方式
- 客户端：Unity打开Client/项目 → 运行
- 服务端：dotnet run --project Server\Jx3.MockServer\Jx3.MockServer.csproj -c Release
- 构建：Unity命令行 batchmode 验证编译
