# 一、项目概述

## 1.1 游戏定位
- **游戏类型**: 英雄收集+ARPG武侠MMORPG (手游/PC双端)
- **世界观**: 剑侠情缘世界观，十二大门派，江湖恩怨
- **目标平台**: iOS / Android / PC (Unity跨平台)
- **核心玩法**: 收集英雄 → 培养阵容 → 副本挑战 → PVP竞技 → 自由交易 → 社交互动

## 1.2 核心玩法说明

### ⭐ 英雄收集玩法 (核心特色)
- 玩家不创建单一角色，而是**收集众多英雄角色**
- 每个英雄有独立的故事背景、技能体系、外观
- 英雄通过：抽卡(招募)、剧情解锁、活动获得、交易行购买等方式获取
- 战斗中可携带3-4名英雄，随时切换操作
- 未上阵的英雄提供羁绊加成

### 英雄与羁绊示例
`
英雄: 李复         | 定位: 近战输出/控制 | 品质: ★★★★★
  背景: 鬼谋遗策传人, 武林智者
  技能: 破军式 / 八卦掌 / 龙啸九天 / 九天惊雷(大招)
  羁绊: 秋叶青(师徒) - 同时上阵攻击+15%
      李承恩(旧识) - 同时上阵生命+10%

## 4.0 模块接口设计规范

每个模块按以下规范定义接口:

`
模块名: XXX
接口前缀: XXX
依赖模块: [模块A, 模块B]
核心数据结构: (Protobuf message定义)
消息列表:
  - CS_XXX_操作名  (客户端→服务器)
  - SC_XXX_操作名  (服务器→客户端)
`

> 详细协议定义见 Documents/接口协议/ 下的 .proto 文件

## 4.1 账号与招募系统 (S01)

### 功能说明
- 账号注册/登录/第三方授权
- 英雄招募(抽卡)系统:
  - 新手池: 首次10连必出5星
  - 常驻池: 所有英雄概率UP
  - 限时池: 特定英雄概率提升
  - 保底机制: 90抽保底5星
- 英雄召唤/重复英雄转化为碎片

### 招募(抽卡)设计

`
招募规则:
  ┌──────────────────────────────────────────────┐
  │ 英雄品质: ★★★★(紫色) / ★★★★★(金色)            │
  │                                               │
  │ 概率:                                         │
  │   ★★★★★: 基础0.6% (含保底)                     │
  │   ★★★★:  5.0%                                │
  │   碎片/材料: 94.4%                             │
  │                                               │
  │ 保底机制:                                      │
  │   - 90抽内必出★★★★★                            │
  │   - 若提前出5星, 保底重置                       │
  │   - 限时池: 50%概率为当期UP英雄                 │
  │                                               │
  │ 重复英雄:                                      │
  │   获得已拥有英雄 → 自动转化为碎片x50             │
  │   英雄升星所需碎片: ★→★x30, ★★→★★★x60        │
  │   ★★★→★★★★x90, ★★★★→★★★★★x120            │
  └──────────────────────────────────────────────┘
`

### 消息定义

`
模块名: Recruit
依赖模块: IPlayer

消息:
  CS_RECRUIT_DRAW            # 抽卡(单抽/十连)
  CS_RECRUIT_POOL_LIST       # 请求卡池信息
  CS_RECRUIT_PITY_INFO       # 请求保底信息
  CS_RECRUIT_EXCHANGE_FRAG   # 碎片兑换英雄

  SC_RECRUIT_DRAW_RESULT     # 抽卡结果
  SC_RECRUIT_POOL_LIST       # 卡池列表
  SC_RECRUIT_PITY_UPDATE     # 保底计数更新
  SC_RECRUIT_NEW_HERO        # 获得新英雄
  SC_RECRUIT_FRAG_UPDATE     # 碎片数量更新

核心数据结构 (proto):
  message RecruitResult {
    repeated HeroItem items = 1;    // 本次抽到的物品
    uint32 pity_count = 2;          // 当前保底计数
    bool is_new_hero = 3;           // 是否新英雄
  }

  message HeroItem {
    uint32 hero_id = 1;
    uint32 quality = 2;             // 4=紫色 5=金色
    uint32 fragment_count = 3;      // 获得碎片数(重复时)
  }
`

## 4.2 英雄系统 (S02)

### 功能说明
⭐ 游戏核心系统

- 英雄收集: 当前拥有所有英雄列表
- 英雄信息: 等级/经验/品质/星级/技能等级
- 英雄培养:
  - 升级: 消耗经验药水/战斗经验
  - 升星: 消耗英雄碎片提升品质
  - 突破: 达到等级上限后突破提升上限
- 英雄详情: 属性面板/技能/背景故事/语音
- 编队系统: 上阵3-4名英雄(1主战+2-3替补)
- 英雄切换: 战斗中随时切换操作角色

### 英雄数据结构

## 4.3 羁绊系统 (S03)

- 英雄间的关系网络
- 羁绊类型: 师徒/挚友/情侣/同门/宿敌

- 羁绊效果: 属性加成/特殊被动
- 羁绊等级: 可升级(消耗羁绊道具)
- 羁绊剧情: 解锁特定英雄对话/剧情

### 羁绊设计示例

羁绊: [师徒情深] — 李复 + 秋叶青

  Lv2 效果: 全队攻击+8%, 李复技能CD-5%
  Lv3 效果: 全队攻击+12%, 李复技能CD-10%, 秋叶青暴击+8%

  类型: 同门
  激活: 拥有其中2人
  Lv1(2人): 全队治疗效果+8%

羁绊: [江湖旧识] — 李复 + 李承恩

`

- **搜索**: 按名称/分类/品质/等级/价格范围

### 交易流程

```
卖家流程:
  1. 打开背包 -> 选中可交易物品
  2. 点击上架拍卖行
  3. 选择分类 -> 设置价格 -> 选择时长
  4. 系统扣上架费(1%) -> 确认上架
  5. 卖出: 金币(扣5%手续费) -> 邮件发送
  6. 到期未卖: 物品邮件退回

买家流程:
  1. 打开拍卖行 -> 搜索/筛选
  2. 查看物品详情 -> 点击购买
  3. 确认支付 -> 物品进背包
```

## B. 货币经济模型

```
产出:
  金币: 任务/副本/日常/出售
  绑元: 充值/月卡/活动
  通宝: 充值(1元=10通宝)

消耗:
  金币: 技能升级/强化/拍卖行购买
  绑元: 商城/抽卡/副本重置
  通宝: 拍卖行/特殊外观/交易

通胀控制:
  1. 交易手续费5%
  2. 装备强化递增消耗
  3. 技能升级消耗
  4. 日常消耗(传送/修理)
  5. 上架费(售价1%)
```
